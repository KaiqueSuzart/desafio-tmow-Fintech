================================================================================
PROMPT PARA RECRIAR O PROJETO "EXTRATOR DE TABELAS HTML" DO ZERO
================================================================================

Cria um projeto Python completo para extrair tabelas HTML de qualquer URL
(focado no Portal da Transparência do Brasil mas funciona com qualquer site).
O projeto tem: scraper (Playwright + BeautifulSoup), API (FastAPI), banco
(SQLite), assistente IA para filtros (OpenAI opcional), e frontend
(HTML/CSS/JS vanilla) — tudo dark mode, moderno, em pt-BR.

================================================================================
FICHEIRO: ai_suggest.py — ASSISTENTE IA DE FILTROS
================================================================================

Objetivo: receber um pedido em linguagem natural em português e devolver
parâmetros de query string + período (YYYY-MM) para preencher o formulário.
Funciona com ou sem OPENAI_API_KEY.

Constantes / tabelas:
- MES_PT: dict nome→número (janeiro=1 … dezembro=12, com variante "marco"=3)
- _VALID_UF: frozenset com as 27 siglas válidas (AC AL AP AM BA … TO)
- _BR_UF_NAME_PAIRS: list de (nome_normalizado, sigla) — todos os 26 estados +
  DF, com variantes sem acento (são paulo / sao paulo, paraná / parana, etc.).
  ORDENAR por comprimento decrescente do nome (evitar que "Rio" case antes de
  "Rio de Janeiro" ou "Rio Grande do Sul")

Funções de período:
- _month_pattern() → regex alternation de todos os nomes de mês (por comprimento)
- parse_periodo_meses_pt(intent) → (YYYY-MM | None, YYYY-MM | None):
  Procura nomes de meses no texto; se houver ano 20xx no texto usa-o, senão
  ano atual. Se 2 meses: retorna (menor, maior). Se 1: retorna (mês, mês).
- _valid_iso_mes(s) → str|None: valida formato "YYYY-MM"
- _intent_has_explicit_year(text) → bool: re.search(r"\b20\d{2}\b")
- _apply_current_year_if_no_explicit_year_in_intent(intent, pde, pat):
  Se o utilizador NÃO escreveu um ano 20xx, substitui o ano nos YYYY-MM pelo
  ano civil (datetime.date.today().year). OBRIGATÓRIO — senão o GPT inventa anos.

Funções de UF / estado brasileiro:
- _fold_for_uf(s) → str: minúsculas + remove acentos + normaliza espaços
- _uf_sigla_from_intent(intent) → str|None:
  1. Procura sigla isolada de 2 letras que pertença a _VALID_UF
  2. Procura nome de estado (mais longo primeiro) via _BR_UF_NAME_PAIRS
- _sigla_from_localidade_value(val) → str|None: se o valor for nome de estado
- _form_field_keys_lower(form_fields) → dict lower(name) → name real
- apply_brazil_uf_rules(params, intent, form_fields) → params corrigido:
  1. Deteta sigla do pedido ou do valor de localidadeGasto
  2. Preenche campo «uf» (ou «estado») com a sigla
  3. Remove localidadeGasto se continha apenas o nome do estado (evita duplicação)

Sanitização:
- _sanitize_params_from_model(raw) → dict[str,str]: ignora None, "None", "null",
  "undefined", "nan", dicts e listas — evita ?campo=None na URL

Heurísticas locais:
- _heuristic_suggest(intent, form_fields, url) → dict params (url necessário para regras Portal):
  - UF: usa _uf_sigla_from_intent para qualquer estado brasileiro (não só SP)
  - Valor numérico (>=4 dígitos): procura campo com "liquidado"+"de" ou
    "empenhado"+"de" e preenche
  - No final: apply_brazil_uf_rules para normalizar UF

Função principal:
- suggest_filters_ai(url, intent, form_fields) → dict (sempre passar url às heurísticas e apply_portal_funcao_rules):
  1. Extrai período por heurística: pde_h, pat_h = parse_periodo_meses_pt(intent)
  2. Prepara compact (amostra de campos para o prompt)
  3. Se OPENAI_API_KEY:
     a. Prompt com: URL, pedido, data de hoje + ano atual, campos, formato JSON
        esperado, regras (usar sigla UF no campo «uf», nunca null como texto,
        ano corrente se não explícito)
     b. System message menciona ano corrente e regra de UF
     c. Modelo: gpt-4o-mini, temperature=0.15, response_format=json_object
     d. Pós-processamento:
        - _sanitize_params_from_model
        - apply_brazil_uf_rules (corrige UF mesmo que o GPT erre)
        - Período: _valid_iso_mes + fallback para heurística
        - _apply_current_year_if_no_explicit_year_in_intent
     e. Erro do OpenAI → fallback heurístico
     f. ImportError (openai não instalado) → fallback + mensagem "pip install openai"
  4. Sem chave: heurísticas locais + período por meses
  5. Retorna SEMPRE: { ok, message, params, periodo_de_mes, periodo_ate_mes, note }

================================================================================
FICHEIRO: scrape_lista.py — SCRAPER (Playwright + BeautifulSoup)
================================================================================

Constantes:
- IFRAME_SELECTOR_AUTO = "__auto__"
- NAV_TIMEOUT_MS: configurável via env var PLAYWRIGHT_NAV_TIMEOUT_MS (default
  120_000, mín 30_000, máx 300_000). Usar em goto e alinhar com defaults do
  context/page. Calcular uma vez no import via _nav_timeout_ms().
- CHROME_USER_AGENT — string de User-Agent real do Chrome (Windows)

Funções utilitárias:
- merge_query_params(url, updates) — atualiza query string; valor None remove o param
- mes_para_de_ate(periodo_de, periodo_ate) — inputs "YYYY-MM" (type month) → query
  de/ate em formato BR dd/mm/aaaa: primeiro dia do mês "de" e último dia do mês "até"
  (ex.: 2024-01..2024-01 → 01/01/2024 e 31/01/2024). ERRO COMUM: gerar MM/DD/aaaa
  ou trocar dia com mês — invalida o filtro no Portal.

Extração BeautifulSoup (parsing puro, sem browser):
- _row_cells(tr) — texto de cada th/td filho direto
- _unique_headers(labels) — garante nomes únicos (adiciona " #2" etc)
- _is_nested_table(table) — True se table está dentro de outra table
- extrair_uma_tabela_generica(table, skip_first_column) — de um Tag <table>,
  extrai (headers, rows). Suporta com e sem <thead>. Se skip_first_column=True,
  remove a 1.ª coluna (ex.: coluna "Detalhar" do Portal)
- extrair_todas_tabelas_top_level(html, skip_first_column) — todas as <table>
  de nível superior (ignora aninhadas), devolve lista de dicts com:
  index, element_id, class, headers, rows, row_count

REGRAS CRÍTICAS do parsing:
1. Tabelas aninhadas (table dentro de table) devem ser IGNORADAS
2. Se não há <thead>, o 1.º <tr> vira cabeçalho
3. Usar zip_longest(headers, vals, fillvalue="") para linhas com colunas a mais/menos
4. Cabeçalhos vazios viram "coluna"
5. Cabeçalhos duplicados recebem sufixo " #2", " #3" etc

Navegação Playwright:
- _apply_timeouts_page(page) — page.set_default_timeout(NAV_TIMEOUT_MS) e
  set_default_navigation_timeout(NAV_TIMEOUT_MS); chamar antes de qualquer goto
- _root_page_or_iframe(page, iframe_selector, log) — modo manual: espera o
  seletor CSS, pega element_handle → content_frame()

DETEÇÃO AUTOMÁTICA DE IFRAME (muito importante — sem isto W3Schools e outros falham):
- _enumerate_frames_in_subtree(page, subtree_root) — usa page.frames (Playwright
  expõe TODOS os frames incluindo aninhados) e filtra pela sub-árvore
- _try_frames_for_selector(frames, ws, timeout_ms, log) — tenta wait_for_selector
  em cada frame; devolve o primeiro que tiver o seletor. Captura tanto
  PlaywrightTimeoutError como Exception genérica (cross-origin etc)
- _root_page_auto_iframe(page, ws, log, prefer_iframes_first) — com page.frames:
  separa main (frames[0]) dos outros; se prefer_iframes_first, testa outros
  primeiro e main depois; timeout dinâmico = max(10_000, min(25_000, 180_000 // n))

ATALHO W3SCHOOLS TRYIT (sem isto o Tryit nunca funciona!):
- _try_w3schools_tryit_iframe_result(page, ws, log):
  1. Espera #iframeResult (60s)
  2. Executa via page.evaluate: window.editor.save() + submitTryit(1)
     (o Tryit usa CodeMirror + document.write, o iframe fica vazio sem isto)
  3. Pega content_frame() do #iframeResult
  4. Espera ws dentro do frame (90s)
  5. Devolve (frame, True) ou (None, False)

- _resolve_scrape_root(page, iframe_selector, ws, log, prefer_iframes_first):
  Devolve (root_document, já_encontrou_ws). Se "__auto__" → auto-iframe.
  Se string CSS → modo manual. Se vazio → page.

- _goto_pagina_e_html(page, url, wait_selector, ..., iframe_selector):
  0. _apply_timeouts_page(page) PRIMEIRO (evita timeout 30000ms default no goto)
  1. wait_until no primeiro goto:
     - W3Schools Tryit → "load" (iframe/editor)
     - Portal da Transparência (host) ou wait_selector #lista → "domcontentloaded"
       (NÃO usar "load" no Portal: muitos recursos/analytics impedem o evento "load"
       ou demoram demais → Page.goto timeout mesmo com 90s em alguns ambientes)
     - resto → "domcontentloaded"
  2. page.goto(url, wait_until=..., timeout=NAV_TIMEOUT_MS)
  3. Se URL é W3Schools Tryit → tenta atalho _try_w3schools_tryit_iframe_result
  4. Se não encontrou → _resolve_scrape_root (auto ou manual)
  5. Se ainda não achou ws → _enumerate_frames_in_subtree + _try_frames_for_selector
  6. Se AINDA não → root.wait_for_selector(ws, 90s) com mensagem de erro descritiva
  7. Se #lista → estabilização extra: networkidle + esperar linhas no tbody
  8. Senão → espera rápida por <table> (15s) + networkidle opcional
  9. Devolve root.content()

Funções principais de extração:
- buscar_tabelas_url(url, wait_selector, table_index, prefer_table_id,
    skip_first_column, headed, wait_network_idle, log, iframe_selector):
  Abre browser headless, vai à URL, extrai HTML, fecha browser,
  parse com BS4. Devolve (todas_tabelas, headers_escolhida, rows_escolhida).
  IMPORTANTE: set_default_timeout(NAV_TIMEOUT_MS) no context E na page.
  set_default_navigation_timeout(NAV_TIMEOUT_MS) também.

- buscar_tabelas_url_paginada(url_base, wait_selector, table_index, ...,
    tamanho_pagina, max_paginas, iframe_selector):
  Loop de paginação. Para o Portal usa pagina=1,2,3... Para outros usa offset.
  NÃO parar só porque len(linhas) < tamanho_pagina — a próxima página vazia
  é que encerra (o HTML às vezes perde 1 linha e isso parava antes da pág 2).
  Cada linha recebe _pagina e _linha (contador global).
  Só adiciona paginacaoSimples/tamanhoPagina se for Portal.

- probe_pagina(url, wait_selector, iframe_selector, log):
  Navega e extrai: form_fields (name, type, label, options), tables (resumo),
  url_final, table_count. Usado pelo endpoint POST /api/probe.

REGRAS CRÍTICAS do Playwright:
1. NAV_TIMEOUT_MS=120000 (configurável); usar no page.goto e nos set_default_*
2. SEMPRE set_default_timeout / set_default_navigation_timeout no context E page
3. SEMPRE _apply_timeouts_page(page) no início de _goto_pagina_e_html antes do goto
4. User-Agent real do Chrome (sem isto sites bloqueiam)
5. --disable-blink-features=AutomationControlled no launch
6. locale="pt-BR" no contexto
7. viewport 1365x900
8. Accept-Language: pt-BR
9. Capturar PlaywrightTimeoutError como falha normal (ValueError), não crash

  - Valida período (ambos ou nenhum, de <= ate)

================================================================================
ARMADILHAS / BUGS QUE FORAM RESOLVIDOS (NÃO REPETIR!)
================================================================================

1. TIMEOUT DEFAULT DO PLAYWRIGHT
   O Playwright tem timeout de 30s por defeito. OBRIGATÓRIO:
   - NAV_TIMEOUT_MS = 120_000 (configurável via env) e usar no
     page.goto(..., timeout=NAV_TIMEOUT_MS)
   - context.set_default_timeout(NAV_TIMEOUT_MS) e
     context.set_default_navigation_timeout(NAV_TIMEOUT_MS)
   - page.set_default_timeout(NAV_TIMEOUT_MS) e
     page.set_default_navigation_timeout(NAV_TIMEOUT_MS)
   - Função _apply_timeouts_page(page) chamada NO INÍCIO de _goto_pagina_e_html
     ANTES do primeiro page.goto (senão o erro mostra "30000ms exceeded").
   - Portal: primeiro goto com wait_until="domcontentloaded", não "load" (ver
     armadilha 11).

2. PAGINAÇÃO DO PORTAL NOUTROS SITES
   Os params paginacaoSimples/tamanhoPagina/offset/pagina SÓ devem ser
   adicionados à URL quando o host é portaldatransparencia.gov.br.
   Noutros sites corrompem o pedido → 404 ou HTML errado.

3. IFRAME SEM # (ex: iframeResult)
   O utilizador pode escrever "iframeResult" em vez de "#iframeResult".
   _normalize_iframe_selector deve prefixar # quando parece um id CSS.

4. W3SCHOOLS TRYIT — document.write
   O preview do Tryit não carrega por URL. O JS da página cria um iframe
   vazio (#iframeResult) e usa submitTryit() + document.write() para meter
   o HTML do editor dentro. Sem executar submitTryit via page.evaluate,
   o iframe fica vazio e o seletor "table" nunca aparece → timeout.
   OBRIGATÓRIO: editor.save() + submitTryit(1) antes de procurar.

5. IFRAMES ANINHADOS
   Usar page.frames (lista achatada de TODOS os frames, incluindo aninhados)
   em vez de page.locator("iframe") que só vê nível 1.

6. PARAR PAGINAÇÃO SÓ POR len < tamanho
   NÃO parar a paginação só porque uma página devolveu menos linhas que
   tamanho_pagina. O HTML às vezes perde 1 linha de layout. A próxima
   página VAZIA é que encerra o ciclo.

7. PORTAL #lista — ESTABILIZAÇÃO AJAX
   Após navegar para nova página no Portal, o #lista esvazia e reenche
   via AJAX. Sem esperar (networkidle + linhas no tbody), page.content()
   captura HTML sem dados → paginação para na 2.ª página.

8. CACHE DO app.js (304)
   Incluir ?v=XXXXXXXX no <script src> para forçar atualização após mudanças.

9. ERRO 502 vs 422
   Usar 502 para falhas de extração (timeout, sem tabela, iframe bloqueado).
   422 no browser parece "erro de formulário" e confunde o utilizador.

10. PORTAL USA pagina=1,2,3 (NÃO offset)
    No portaldatransparencia.gov.br, o parâmetro que avança resultados é
    "pagina" e não "offset". Offset deve ser removido (None).

11. PRIMEIRO GOTO NO PORTAL — NÃO USAR wait_until="load"
    Sites públicos (Portal) carregam muitos recursos; o evento "load" pode não
    disparar a tempo ou segurar o goto. Usar "domcontentloaded" no primeiro
    page.goto e depois esperar #lista + estabilização (networkidle + linhas no tbody).
    Manter wait_until="load" apenas para W3Schools Tryit se necessário.

12. FILTRO DE PERÍODO (de/ate) — FORMATO BR dd/mm/aaaa
    mes_para_de_ate deve gerar strings "01/MM/AAAA" e "último_dia/MM/AAAA".
    Se inverter dia/mês ou usar formato errado, o Portal ignora o filtro e parece
    que os inputs "não fazem nada".

13. FRONTEND — syncUrlDerivedOptions SÓ NO blur
    Se a URL default já vier preenchida, sem chamar syncUrlDerivedOptions() no
    bind() e no "change" da URL, checkboxes (paginação, skip coluna) e preset #lista
    podem ficar desalinhados com o que o utilizador espera ao clicar "Carregar".

14. CSS DA PRÉ-VISUALIZAÇÃO — "TUDO NA VERTICAL"
    Evitar table-layout:fixed + quebra de palavra em grelhas com muitas colunas:
    as linhas ficam altas e não parecem lista. Preferir scroll horizontal +
    white-space: nowrap nas células (estilo planilha).

15. UnboundLocalError em buscar_tabelas_url (table_index)
    Dentro de uma função interna (ex.: run()), NÃO fazer "table_index = i" se
    "table_index" for também o nome do parâmetro da função externa: em Python
    isso torna table_index local a run() inteira; se o bloco if prefer_table_id
    não atribuir, table_index fica indefinido e explode ao comparar índices.
    Usar variável local distinta (ti = table_index) e só depois ajustar ti.

16. W3Schools Tryit — primeira tabela com 0 linhas de dados
    Em exemplos HTML, a 1.ª <table> pode ser só uma linha (tratada como cabeçalho),
    logo row_count=0. O utilizador vê "OK — 0 linhas". Solução: após escolher ti,
    se row_count==0 e existir outra tabela com linhas, avançar para a primeira com
    dados; devolver também o índice efetivo (ex.: tuplo com ti) para a API marcar
    selected_table_index correto. No frontend, explicar que período/paginação não
    aplicam ao Tryit.

17. GPT INVENTA ANO (ex. 2023 QUANDO ESTAMOS EM 2026)
    Quando o utilizador diz "janeiro a março" sem especificar ano, o GPT pode
    devolver 2023-01. OBRIGATÓRIO: _apply_current_year_if_no_explicit_year_in_intent()
    substitui o ano pelo ano civil se o pedido não contiver um 20xx explícito.

18. PARAMS COM VALOR None / "None" NA URL
    O GPT pode devolver {"valorEmpenhadoAte": null}. str(None) → "None" na URL,
    que o Portal não reconhece. OBRIGATÓRIO: _sanitize_params_from_model() e
    _sanitize_extra_query() filtram estes valores antes de merge na URL.

19. LOCALIDADE vs UF — DUPLICAÇÃO DE FILTRO
    Se o utilizador pede "Rio de Janeiro", o GPT pode pôr
    localidadeGasto="Rio de Janeiro" em vez de uf="RJ". A API aceita ambos mas o
    portal filtra melhor por UF. OBRIGATÓRIO: apply_brazil_uf_rules() detecta o
    estado e preenche «uf» com sigla, removendo localidadeGasto se for redundante.

20. PERÍODO NO FRONTEND — INPUTS type=month NÃO PREENCHIDOS PELA IA
    O endpoint /api/ai/suggest-filters devolve periodo_de_mes e periodo_ate_mes
    (YYYY-MM). O JavaScript DEVE copiar estes valores para el["periodo-de"].value
    e el["periodo-ate"].value, senão o período não vai no body do /api/scrape e
    o utilizador acha que "não funciona".

21. .env.example SEM CHAVES REAIS + .gitignore
    NUNCA incluir API keys reais no .env.example (risco de vazamento). Usar
    OPENAI_API_KEY= vazio e comentário a apontar platform.openai.com. Criar
    .gitignore com .env, __pycache__, venv.

22. PROBE — PARÂMETROS DINÂMICOS (modal)
    Não usar atributo HTML "id" como nome de parâmetro quando "name" está vazio
    (gera lixo tipo token-input-orgaos, check-group-1). O probe DEVE:
    - Devolver "query_params" = dict parseado da query string da url_final.
    - Só incluir em form_fields inputs/selects com atributo "name" não vazio.
    - Excluir names que comecem por "token-input-" e names "check-group-N".
    O frontend preenche primeiro query_params (valores reais), depois acrescenta
    campos de formulário ainda não listados com valor "".
    Opcional UX: coluna "Parâmetro" readonly para linhas vindas do probe; só
    "Adicionar parâmetro" permite editar o nome da chave.

23. TAMANHOPAGINA: extra_query vs CAMPO DO FORMULÁRIO
    Ao fundir paginação do Portal, se extra_query tiver tamanhoPagina (ou
    tamanho_pagina), esse valor TEM PRIORIDADE sobre o número "Linhas por página"
    do body JSON — senão o utilizador põe 10 na grelha e o backend sobrescreve com 50.
    _preparar_url_scrape pode devolver (url_preparada, tamanho_efetivo). A resposta
    /api/scrape devolve tamanho_pagina = valor efetivo. O frontend sincroniza o
    input "Linhas por página" após probe (query), após IA e após scrape.

24. MÚLTIPLAS PÁGINAS SEM SÓ O CHECKBOX "BUSCAR TODAS"
    use_paginada NÃO pode ser apenas buscar_todas_paginas AND portal — o utilizador
    espera que "Máx. páginas" = 3 com 10 linhas/página traga até 30 linhas mesmo
    sem marcar "Buscar todas". Regra: Portal + paginacao_portal + (buscar_todas
    OR max_paginas > 1). Default de max_paginas no modelo/UI = 1 para não ir buscar
    20 páginas por engano quando o default antigo era 20.

25. IFRAME MANUAL — SUB-ÁRVORE DE FRAMES
    Após content_frame() do iframe manual, percorrer BFS child_frames a partir
    desse frame até encontrar wait_selector; não assumir que page.frames mal filtrado
    resolve.

26. buscar_tabelas_url — ÍNDICE EFETIVO
    Devolver (todas_tabelas, headers, rows, selected_table_index) para o frontend
    e API ficarem alinhados com fallback quando row_count==0.

27. FRONTEND — el[] COMPLETO NO bind()
    Garantir que todos os elementos referenciados (btn-scrape, btn-probe,
    save-title, save-only-filtered, etc.) são atribuídos a el no bind() — senão
    ReferenceError ao submeter.

28. FastAPI detail em ERROS
    O campo "detail" pode ser string ou lista de objetos; o fetch no JS deve
    normalizar (apiErr) para mensagem legível.

29. IA / OPENAI — CHAVE E MENSAGENS
    Reler .env em cada pedido a suggest_filters (load_dotenv(..., override=True))
    ou documentar reinício do Uvicorn após mudar .env. Tratar 401 com mensagem
    curta em PT (não despejar o JSON inteiro no utilizador). Heurística + GPT:
    Portal despesas — "área da saúde" / "saude" → funcaoSubfuncao=FN10 (e regras
    semelhantes para outras funções); instruir o GPT explicitamente no prompt.

30. MODAL CSS — NÃO MISTURAR form-grid COM CADA LINHA DE PARÂMETRO
    Cada linha (Parâmetro | Valor | Remover) deve ser um bloco próprio (cartão ou
    grid local); não pôr form-grid no contentor pai de muitas linhas — vira grelha
    de 4 colunas ilegível.
